- `-t T` 単体で turn ごとの思考サマリが全件出力された。ターン境界と時刻が明確で流れを追いやすい。
- `--width 9999` により折り返しなし。長いサマリも1ブロックに収まる。
- `--md=source` により Markdown コードブロックにならず、ターン境界の `---` がそのまま読めた。ただし箇条書きの `-` がそのまま出るので、ソースとして処理するには都合がよい。
- セッション全体は turn2~5 (ターン番号はインクリメント)。実作業ターンは 2 (方針議論)、3 (cwd/resume 実装)、4 (複数 range 実装)、5 (-m オプション実装) の 4 フェーズ。
- file-ops で実際に Edit されたファイルリストを照合することで、どのターンで何のファイルが触られたか補完できた。timeline `-t T` だけでは「何を実装したか」の概要は追えるが「どのファイルが変わったか」は見えない点が補完のポイント。
