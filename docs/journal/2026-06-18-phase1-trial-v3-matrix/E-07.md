## 手順メモ (条件 E の操作)

```bash
# timeline
claude-session-analysis timeline 66b917d2-43b1-4d3d-ab3d-2637200fadfb \
  -t R --width 9999 --md=source

# file-ops (条件共通)
claude-session-analysis file-ops -d 1 66b917d2-43b1-4d3d-ab3d-2637200fadfb
```

- `-t R` = アシスタント発話のみ。全 9 turn、各 turn の result 要約が末尾に付く
- `--width 9999` = 折り返しなし。コードブロックや表が 1 行で出てくるので索引作成向き
- `--md=source` = Markdown をそのまま出力。コードフェンスや表が崩れずに読める
- file-ops は Edit/Read のパス + turn 番号が JSONL で出る。どの turn でどのファイルを触ったかの骨格把握に有効。ツール引数や内容は含まない
- result 要約が各アシスタント turn に付いているため、「何をしたか」の大枠は timeline だけでほぼ追える。ツール詳細が欲しい場合は `-t RU` 等に切り替えが必要
