## 迷った (判断を要した分岐)

- **DR-0015 の送信 semantics (broadcast vs queue)**: 当初 broadcast を default にする案も出たが、最終的にキュー方式 (取った者勝ち) を default とし、broadcast は明示的な別コマンドに分離した。複数 alive peer への配信時の競合解決が主な論点だった。
- **label vs tag の用語統一**: DR-0015 内で tag (k=v 構造) と label (単純文字列) の使い分けを議論した結果、label が包含的に正しいとして統一。`:` を避けて `=` をセパレータに採用 (Windows の `:` ファイル名禁止が決定打)。
- **cmux-msg user skills の配置 (skills/ vs commands/)**: 当初 `skills/` に `disable-model-invocation: true` で置いていたが、commands/ に置くと `/cmux-msg:<sub>` 形式で名前空間が自然に見えるため移行した。
- **model 指定の安全戦略**: haiku → sonnet[1m] → inherit → context: fork + haiku の順に変遷した。sonnet (200K) + 1M session で credit エラーが実際に発生したことで、context: fork による fresh context 使用が必須と確定した。
- **claude-plugin-reference への追記作業**: 知見を反映しようと bump-version を複数回実行したが、origin より遅れたローカルで連続 bump するミスが発生。abandon → op restore で撤退し、working copy 保持 + push 不要の方針に落ち着いた。
