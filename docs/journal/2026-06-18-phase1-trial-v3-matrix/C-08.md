### `-t TR --width 9999` で得られた情報量の評価

- T列 (思考サマリ) がターンごとの「なぜそうしたか」を伝えてくれるため、R列 (result) だけでは見えない判断経緯が読める
- width 9999 で折り返しがなく、1行=1エントリとして構造的に読みやすい
- md なしでも T/R の接頭辞と turn 番号で十分に構造を把握できた
- **欠けるもの**: 実際のコマンド出力・ツール入力 (`-t F` や `-t B` に相当する情報) がないため、「何が失敗したか」の詳細を T列サマリの記述に頼ることになる。サマリが十分詳細に書かれていれば問題ないが、端折られていると事実確認が困難
