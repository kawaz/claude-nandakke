## 解釈に迷った点

**T25〜27 の「申し送り」の出典。**

R034115dd に「申し送りに従って C6 と C9 を実装」とあるが、`-t R` の範囲内では申し送り元セッションが何かを特定できなかった。T21 が nonstop 宣言で自律進行している旨 (Rb5be7736) があり、複数セッションをまたいでいる可能性がある。

**C8 の最終形。**

C8 は途中で「1password-error-notification の authsock-warden 上流還元 + rule 縮退」として始まり、kawaz 訂正で「Note 削除 + authsock-warden issue 取消」と変化し、最終的に「`1Password` → `ワンパスワード` 1 行変更のみ」に縮退したとある。ただし各段階でどこまで変更が実際にファイルに入ったかは、AI 応答のサマリと file-ops ログを突き合わせないと追えなかった。file-ops ログでは T21 に `1password-error-notification.md` の Read + Write (2 回) が記録されており実際に変更は入っている。索引では「推定」にしたのは最終形の確認を机上でしかしていないため。

**4/4 合議の信頼度。**

索引テーブルでは「4 並列レビュー」の `確信` を「事実」にしているが、レビュー内容の正確性は別問題。kawaz T7 の指摘（同一バイアス 4 回サンプリング）は「レビューが起きた」という事実とは別に「レビューの結果が正しい」という確信に疑義を呈したもので、索引の「事実/推定」列ではこの区別が表現しにくかった。
