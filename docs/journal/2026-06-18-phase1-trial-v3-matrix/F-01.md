## この条件で見えなかったこと

**T (Assistant 思考) + R (Result / ツール結果) を `--md=source` で読む条件。R 型が加わることで A 条件 (T のみ) より大幅に情報量が増えたが、それでも取りこぼした情報がある。**

1. **ユーザの実際の発言原文** (U 型)
   - T 型の「ユーザが〜と言っている」は Claude の解釈要約であり、ユーザ原文は見えない。
   - R 型にもユーザ発言は乗らない。U 型を含めない限りユーザ視点は間接的にしか把握できない。
   - 例: turn 2 で「`/cd` で jsonl の保存パスも変わる」という指摘、turn 5 で `-m N` の要求が突然出てくるが、ユーザがどういう言葉で・どのタイミングで問題を説明したかは T 型の解釈文からしか読めない。

2. **W 型 (会話の流れ / 実際の assistant 応答テキスト)**
   - T 型は「これを実装しようと思う」という内部思考、R 型は「ツール実行の生結果」であり、Claude がユーザに返した応答文 (W 型) は含まれない。
   - R 型の `R...` エントリは本条件では `--md=source` により Markdown のソース状態で出力されるため、読みやすさは確保される。ただし W 型 (会話本文) がない分、「Claude がユーザにどう報告したか」の文面は確認できない。

3. **B 型 (Bash コマンドの実行コマンド文字列)**
   - 本条件では T と R しか含まない。Bash ツールで何のコマンドを叩いたかの詳細 (B 型相当) は見えない。
   - R 型でコマンド実行結果のテキストは出るが、「何を叩いたか」は T 型の文脈から推測するしかない。

4. **F 型 (ファイル操作の詳細差分)**
   - file-ops (`-d 1`) は条件共通で 1 度叩いた。これにより「Edit/Read した」ファイル一覧は把握できる。しかし具体的な変更内容 (before/after) は R 型の出力テキストから読む必要がある。file-ops が補完材料として有効だった点は A 条件と同じ。

## 解釈に迷った点

1. **R 型と T 型の重複情報の扱い**
   - T 型「次に filter.ts に `filterByRanges` を追加」と R 型「Edit: filter.ts」が対応するが、R 型には変更後のテキストが含まれていない場合もある (Edit の R 型は変更結果を全文出力しないケースがある)。どちらを「最終言及」とするかが微妙。今回は T 型の記述を優先した。

2. **--md=source の実際の効果**
   - 出力が大きく (31.3KB) なり、persisted-output ファイルに保存された。内容は Markdown のソース状態 (レンダリングなし) で読めた。テーブルや箇条書きはそのまま `|` や `-` で読めるため、実用上は十分な可読性があった。A 条件 (`--md` なし) との差が出るのは R 型の Markdown コンテンツの多いセッションだが、本セッションの R 型は主に短いコード片や要約なのであまり差は出なかった印象。

3. **「push の可否確認」の相**
   - `T93fe5bdc` (turn 3 末尾) に「push の可否を確認するのが適切」とあり、直後の R 型エントリ (`R` 始まり) に「push (bump 含む) は指示待ち」という文面がある。この R 型が Claude の報告文なのか思考の続きなのか、本条件では W 型がないため「Claude がユーザにどう表明したか」の確認が難しい。

4. **turn 5 の `-m N` 実装の発端**
   - turn 4→5 の間の会話 (U 型) がないため、いつ・どのような文脈でユーザが `-m N` を要求したかが不明。T 型 (`Tbaf2f5d2`) で「2 つの問題を確認する必要がある」という記述から turn 4 の要求が来た後、turn 5 では `-m N` が突然追加議題として現れる。T 型のみでは発端のターンが確定できない。

5. **`--md=source` と A 条件の比較での索引精度差**
   - 本条件 (TR + md=source) の索引テーブルは、A 条件 (T のみ) と topic 数・内容ともほぼ同じになった。R 型を追加しても「新たな topic が発掘される」というよりは「T 型で言及された topic の裏付け情報が増える」という使い方になった。索引精度向上の効果より、R 型による confirmation 効果の方が大きかった。
